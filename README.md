# 資安漏洞原理演示：CSRF 與 XSS (5分鐘演示腳本)

## 0. 開場白 (30秒)
**[講者]**：
大家好，我們這組要介紹的是網站開發中，最經典的兩個資安陷阱。我們不談複雜的程式碼，直接用 **「模擬情境」** 讓大家看懂駭客是怎麼騙過你的瀏覽器的。我們要介紹的是 **CSRF** 和 **XSS**。

---

## 1. CSRF (跨站請求偽造) - 「借刀殺人」 (2分鐘)
*(畫面：開啟 [csrf_demo.html](https://katewayc.github.io/cyber-security-class-scu/csrf_demo.html))*

**[講者]**：
首先是 CSRF。大家可以把它想像成 **「借刀殺人」**。
大家都有網銀吧？當你登入網銀後，瀏覽器會幫你保管一張「VIP 通行證」（也就是 Cookie）。只要有這張證，銀行就認得你是誰。

**[演示動作 1]**：指向畫面上方，確認目前是「紅色 HIGH Risk」狀態。
**[講者]**：
* **左邊**是駭客做的惡意網站。
* **中間**是你的電腦（帶著橘色通行證）。
* **右邊**是銀行伺服器。

**[演示動作 2]**：按下 **「模擬攻擊流程」** 按鈕。
**[講者]**：
今天你不小心點了一個奇怪的連結（左邊）。大家注意看紅色的線！駭客並沒有偷走你的通行證，而是**指揮你的瀏覽器**去銀行轉帳。
銀行看到是你拿著通行證來的（橘色點點），以為是你本人要轉帳，於是錢就被轉走了！這就是 CSRF——駭客不用偷密碼，他直接**控制你的手**去按按鈕。

**[演示動作 3]**：按下 **「開啟 CSRF 防護 (Token)」**，變為綠色安全狀態。
**[講者]**：
修復方法很簡單：銀行除了認通行證，還要多檢查一張 **「隨機號碼牌 (Token)」** 。這張號碼牌駭客拿不到。

**[演示動作 4]**：再次按下 **「模擬攻擊流程」**。
**[講者]**：
看！這次銀行發現：『欸？你有通行證，但號碼牌呢？』於是拒絕了交易，攻擊失敗。

---

## 2. XSS (跨站腳本攻擊) - 「詐騙公告欄」 (2分鐘)
*(畫面：點擊左上角 [「前往 XSS 演示」](https://katewayc.github.io/cyber-security-class-scu/xss_demo.html))*

**[講者]**：
接下來是 XSS。如果說 CSRF 是借你的手，那 XSS 就是 **「在公告欄貼毒藥」**。
想像學校有個留言板，大家都可以貼便條紙。

**[演示動作 1]**：按下 **「一般留言」** -> **「送出留言」**。
**[講者]**：
正常情況下，大家都貼「你好」、「早安」，這很安全。

**[演示動作 2]**：按下 **「植入惡意圖片」** -> **「送出留言」**。
**[講者]**：
但如果網站沒有檢查你在寫什麼，駭客就可以貼一張 **「帶有程式指令」** 的便條紙。
大家看！我只是送出一則留言，結果這個留言板直接被我改造成駭客的廣告版面了！

**[演示動作 3]**：按下 **「植入彈跳視窗」** -> **「送出留言」** (彈出 Alert)。
**[講者]**：
更可怕的是這個。只要你們任何一個人 **「看」** 到這則留言，瀏覽器就會執行這段隱形指令，把你們的帳號密碼偷偷傳給駭客。

**[演示動作 4]**：點擊 **「切換成安全模式」** -> 再次按下 **「植入惡意圖片」** 並送出。
**[講者]**：
修復方法就是網站必須要「過濾」。開啟安全模式後，網站會把所有特殊的符號都當作純文字，不會再執行了。

## 3. 結尾 (30秒)
**[講者]**：
總結來說：
1.  **CSRF** 是利用瀏覽器的「自動信任」，借你的權限做壞事。
2.  **XSS** 是利用網站的「盲目信任」，讓駭客有機會執行惡意指令。

修正這兩個漏洞是保護使用者的基本功。我們的報告到此結束，謝謝大家！
